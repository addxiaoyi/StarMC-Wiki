# StarMC Wiki 贡献指南

欢迎参与 StarMC 百科的建设！为了保证文档的一致性与易读性，我们制定了一套标准的 Markdown 贡献流程与格式规范。

## 1. 快速开始

所有的文档都存储在仓库的 `/public/content/wiki/` 目录下。

1. **创建文件**: 在该目录下新建一个 `.md` 文件，文件名即为该页面的 `slug`（访问路径）。
2. **填写元数据**: 在文件顶部使用特定的注释格式填写页面信息（系统将自动识别并更新侧边栏）。
3. **编写内容**: 使用标准 Markdown 语法编写文档。
4. **提交更新**: 推送至远程仓库，系统将自动部署。

## 2. 核心格式规范

为了让系统正确识别页面属性，请务必在文件顶部包含以下元数据块：

```markdown
<!--
TITLE: 页面显示的标题
CATEGORY: 侧边栏分类名称
LAST_UPDATED: YYYY-MM-DD
PARENT: 父级页面 slug (留空则为一级页面)
ICON: 页面图标 (可选)
-->
```

### 参数说明：
- **TITLE**: 页面在浏览器标签和页面顶部显示的大标题。
- **CATEGORY**: 侧边栏的分组名称。如果该分组不存在，系统会自动创建。
- **PARENT**: 用于定义层级关系。例如：如果你想让“基础指令”页面作为“新手入门”的子页面，则在此填入 `intro`。
- **LAST_UPDATED**: 文档的最后修订日期。

## 3. 内容编写建议

### 层级定义
- 使用 `#` 表示一级标题（通常由系统自动生成，MD 内部建议从 `##` 开始）。
- 使用 `##` 表示二级标题。
- 使用 `###` 表示三级标题。

### 交互组件
- **警告/提示**: 使用 `> ` 引用块。
- **代码块**: 使用 ` ``` ` 包裹指令。
- **子页面跳转**: 直接使用 `[链接文字](/wiki/slug)` 形式。

## 4. 获取模板

您可以直接下载并使用我们的 [基础格式模板](/content/wiki/template.md) 进行创作。
